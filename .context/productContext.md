## WorkToken Core Philosophy
**WorkToken** is a platform for organizing, tracking, and incentivizing work within academic and enterprise environments.

- **Purpose**: It aligns faculty performance with tokenized rewards. Faculty members teach scheduled classes (Structured Work) and volunteer for extra projects (Unstructured Work) to earn tokens.
- **The 85% Threshold Goal**: The core gamification loop requires a user to reach an 85% progress threshold (calculated dynamically based on attendance and task completion). Only upon reaching 85% is the user permitted to initiate a fiat salary transfer.
- **Approval Chain**: Salary transfers and unstructured task proofs are subjected to a rigorous, multi-role approval workflow:
  - Faculty submits.
  - HOD reviews and approves.
  - Finance executes the final smart contract release.
- **Value Proposition**: Gamifies daily academic work, provides transparent tracking of tasks, removes administrative bottlenecks, and automates multi-stage approvals using an immutable audit trail.
